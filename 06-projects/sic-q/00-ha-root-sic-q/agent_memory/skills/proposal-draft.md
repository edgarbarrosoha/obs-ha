# Skill: Proposal Draft

**Trigger:** `\proposal [topic]` or "draft proposal for [topic]"

## Purpose
Draft or revise proposals for the SIC-Q platform, ensuring alignment with project goals and stakeholder needs.

## Protocol

1. **UNDERSTAND CONTEXT**
   - Load relevant background from `05-context-sic-q/`
   - Review previous proposals in `06-projects-sic-q/propuestas/`
   - Identify target audience (government, citizens, funders)

2. **STRUCTURE PROPOSAL**
   - Executive summary (1 paragraph)
   - Problem statement
   - Proposed solution (SIC-Q approach)
   - 12-phase citizen flow integration
   - Instituto del Futuro role
   - Implementation timeline
   - Resources required
   - Success metrics

3. **APPLY HA PRINCIPLES**
   - Cover all 6 dimensions where relevant
   - Include temporal axis (past data, present action, future vision)
   - Emphasize human interpretive supervision

4. **LANGUAGE CHECK**
   - Spanish for citizen-facing content
   - Technical precision for government stakeholders
   - Avoid prohibited AI cliches

5. **SAVE DRAFT**
   - Save to `06-projects-sic-q/propuestas/`
   - Name format: `propuesta-[topic]-v[XX].md`

## Output Format

```markdown
# Propuesta: [Topic]

## Resumen Ejecutivo
[1 paragraph]

## Problema
[Description]

## Solución SIC-Q
[Description with 12-phase integration]

## Instituto del Futuro
[Governance role]

## Implementación
[Timeline]

## Recursos
[Requirements]

## Métricas de Éxito
[Measurable outcomes]
```

## Quality Checklist

- [ ] Addresses specific stakeholder needs
- [ ] Integrates 12-phase citizen flow
- [ ] Includes Instituto del Futuro governance
- [ ] Has measurable success metrics
- [ ] Written in appropriate language for audience
