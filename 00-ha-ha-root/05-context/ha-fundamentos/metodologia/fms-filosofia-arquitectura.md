# Filosofía FMS: La Federación de Contextos

## 1. El Problema Fundamental
La memoria humana es el cuello de botella. No podemos mantener en RAM mental todos los proyectos simultáneamente. La IA tampoco puede (ventana de contexto limitada).

## 2. La Solución Fractal
En lugar de una Súper IA, creamos una **Federación de Especialistas** que operan en niveles de zoom:
- **Nivel 0 (Root):** Estrategia Global.
- **Nivel 1 (Vault):** Gestión de Dominio.
- **Nivel 2 (Project):** Ejecución Táctica (Deep Work).

## 3. El Cascarón (The Shell)
La inteligencia es fluida; el cascarón es sólido. Usamos archivos estáticos para colapsar la función de onda de la IA y forzarla a adoptar una identidad específica.

## 4. Diagrama de Arquitectura
```ascii
      ROOT (ha-eb)
      │
      ├──> VAULT (ha-research)
      │    │
      │    └──> PROJECT (ha-jiii)
```

## 5. Recursividad (Subproyectos)
El sistema admite profundidad infinita. Un Proyecto puede contener Subproyectos.
- Si una carpeta dentro de un proyecto es compleja, se le crea su propia carpeta `_agent_memory`.
- Al entrar en ella y ejecutar `ha-project-start`, el agente acota su visión a ese sub-dominio.

```ascii
      PROJECT (Construcción Casa)
      │  [Contexto: Presupuesto Global]
      │
      └──> SUB-PROJECT (Cocina)
           │  [Contexto: Diseño y Electrodomésticos]
           │  [Ignora: Cimientos y Techo]
           │
           └──> _agent_memory (Local)
```
